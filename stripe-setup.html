公開と決済のセットアップ

Netlify（公開）＋ Stripe（決済）方式。決済リンクを手作業で6本作る必要はありません。

この方式のいいところ：購入ボタンが押されると、サーバー関数がその商品の価格で決済ページを自動生成します。だから Stripeでやるのは「キーを1個コピーして貼る」だけ。新商品もサイトの商品リストに1行足すだけで対応できます。
◯ フェーズ1：Netlifyで公開する
1

Netlify に登録（無料）

app.netlify.com/signup で「GitHubで続ける」を選ぶと、いまのGitHubアカウントですぐ登録できます。

2

リポジトリをつないで公開

「Add new site（サイトを追加）」→「Import an existing project」→ GitHub → gyosei-pottery を選択。設定はそのまま「Deploy」。

数十秒で https://ランダム名.netlify.app の公開URLが発行されます。あとで好きな名前や独自ドメインに変更できます。
◯ フェーズ2：Stripeをつなぐ（あとでOK）
3

Stripe に登録（無料）

dashboard.stripe.com/register で登録。本人確認（氏名・住所・振込先口座）を済ませます。

初期費用 ¥0
月額 ¥0
手数料 3.6%
4

シークレットキーをコピー

Stripe管理画面 →「開発者」→「APIキー」→ 「シークレットキー」（sk_live_...）をコピー。

シークレットキーはパスワードと同じくらい大事です。人に見せたりチャットに貼ったりしないこと。貼るのは次の「Netlifyの環境変数」だけ。
5

Netlifyにキーを登録して完了

Netlifyのサイト →「Site configuration」→「Environment variables」→「Add a variable」で、

  • キー名：STRIPE_SECRET_KEY
  • 値：さっきコピーした sk_live_...

保存したら「Deploys」→「Trigger deploy」で再公開。これで購入ボタンが本物の決済につながり、買える状態の完成です。

◯ フェーズ3：仕上げ
6

特定商取引法ページを記入

tokushoho.html のオレンジ色の箇所（氏名・住所・電話）を実際の情報に。ネット販売では法律で必須です。私（Claude）が記入・公開まで対応します。

あなたがやるのは：①Netlify登録＋接続 → ②Stripe登録 → ③キーをNetlifyに貼る、の3つだけ。
キーを貼るまでは購入ボタンは「販売準備中です」と表示され、誤購入は起きません。フェーズ2は後回しでもOKです。
